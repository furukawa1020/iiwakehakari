AIの判断に、理由の重さで交渉する卓上デバイス

あなたの言い訳、今日は何グラム？

AIが人間を判断する時代に、人間はAIへ異議申し立てできるのか。 いいわけ秤は、従うか無視するかの間にある 「交渉の余白」を、必要な瞬間にだけ現れるUIとして物体化する。

01 / Problem

AIは、私たちの行動を判断するようになった

集中支援アプリ、習慣化アプリ、スマートホーム、ウェルビーイング支援。 AIやスマートデバイスは、私たちの状態を推定し、行動を促すだけでなく、 ときに止めるようになっている。

「今はやめた方がいい」

「そろそろ行動した方がいい」

「その選択は望ましくない」

問題は、AIが判断すること自体ではない。判断された側の人間が、 その判断にどう返答できるのかである。

止まれ
条件付き
通ってよい
便利な判断は、いつの間にか一方通行の信号になる。

02 / HCI Lineage

CUIからGUI、Zero UIへ。そしてNegotiable UIへ

UIは、操作の道具であると同時に、ユーザーがシステムへ要求し、 修正し、異議申し立てするための足場でもあった。

CUI / DOS

命令するUI

ユーザーはコマンドを打ち、システムに明示的に要求する。

GUI

選択するUI

ボタン、窓、メニューを通じて、操作の選択肢が見える。

Zero UI

消えるUI

操作の手間は減るが、判断の根拠や訂正の入口も見えにくくなる。

Negotiable UI

交渉できるUI

普段は黙っていて、ズレた瞬間だけ理由を返す場を開く。

Zero UIが消したのは、操作の手間だけではない。 ユーザーが異議申し立てするための足場でもある。

03 / Human Context

人間には、理由がある

行動変容システムでは、先延ばしや逸脱は失敗として扱われがちである。 しかし実際には、その場の人間には文脈がある。

疲れている。
あと5分だけならできる。
今日は例外かもしれない。
タスクの定義が大きすぎる。
これはサボりではなく回復かもしれない。
AIの判断が、今の文脈とズレているかもしれない。

AIの判断に従うか、無視するか。その二択だけではなく、 理由を返して交渉する余地が必要ではないか。

04 / Concept

言い訳を、交渉材料にする

いいわけ秤は、AIが止めた行動に対して、ユーザーが理由を提示して 交渉する装置である。必要のないときは静かに黙り、AIの判断と 人間の文脈がズレた瞬間だけ、理由を返すUIとして立ち上がる。

普通のシステム 行動 → 判定 → 許可 / 不許可
いいわけ秤 行動 → 判定 → 理由提示 → 交渉 → 条件付き判断
理由の重さ 62g

黄：10分だけ条件付き許可

05 / Experience

体験の流れ

来場者は自分の行動と言い訳を選び、それがAIにどの程度通るかを見る。 「止められた人」ではなく、「理由を返せる人」になる体験である。

1

行動カードを選ぶ

「あと5分だけSNSを見る」「締切前だけど休憩する」

2

AIが初期判定する

赤：止まれ / 黄：条件付き / 青：通ってよい

3

理由カードを載せる

「疲れている」「研究に必要」「今日だけ」「もう少しで終わる」

4

物理的に返す

針が傾き、LEDが変わり、ゲートが開く / 半開き / 閉じる。

06 / Form

なぜ「秤」なのか

交渉とは、理由の重みづけである。同じ言葉でも、その重さは人、 状況、履歴によって変化する。

本当に限界のときの「疲れている」は重い。毎回何かを避けるための 「疲れている」は、だんだん軽くなるかもしれない。

1回目の「疲れている」 重い
毎回の「疲れている」 軽い
区切りをつける「あと5分」 重い
毎回30分になる「あと5分」 軽い

07 / System

システム構成

入力は行動カードと理由カード。Solist-AIが初期判定と理由の重み推定を行い、 針・LED・ゲートのリアルタイム制御へ返す。

行動カード

あと5分SNS / 締切前の休憩 / 今日だけサボる

カード検知

RFID / 抵抗値 / ボタン選択

Solist-AI™

初期判定、理由の重み推定、交渉パターンのオンデバイス学習

物理出力

サーボ針、赤黄青LED、小型ゲート、ブザー、短文表示

交渉入力

理由カード、猶予時間ダイヤル、異議申し立てボタン、例外申請レバー

最小実装：ボタン選択 + サーボ + LED 発展実装：RFIDカード + ダイヤル + レバー 出力：針角度 / ゲート開度 / 短文フィードバック

09 / Solist-AI

Solist-AI™らしい点

この作品では、個人的な理由や逸脱の履歴を手元に閉じ、 その人ごとの交渉パターンを学習することが重要になる。

スタンドアローンAI動作

言い訳や逸脱の履歴をクラウドに送らず、手元で処理する。

オンデバイス学習・追加学習

理由カードの選択、操作パターン、交渉結果を学習し、重みづけを変化させる。

AI処理以外の処理を同時実行

推論結果に応じて、針・LED・小型ゲートをリアルタイムに制御する。

選んだ理由 操作の仕方 交渉結果 次回の重み

Conclusion

AIに従う / 無視する、の二択を超える

いいわけ秤が作りたいのは、AIが人間をより強く管理する未来ではない。 普段は静かに環境へ溶け込み、ズレが生まれた瞬間だけ交渉の場を開く。 UIが消えた先に、もう一度、人間がシステムへ理由を返せる入口を作る。

AIに判断された人間が、もう一度自分の理由を返せる未来へ。
